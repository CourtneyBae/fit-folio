# PRD
## 포트폴리오-JD 핏 진단 서비스 MVP

---

# 1. 개요

## 한 줄 정의

프로덕트 디자이너가 지원 전 자신의 포트폴리오가 JD와 얼마나 맞는지 빠르게 진단하고, 수정 우선순위를 제안받는 서비스

---

# 2. 제품 목표

## 제품 목표

- 사용자가 포트폴리오와 JD 간의 간극을 빠르게 파악할 수 있도록 돕는다.
- 지원 직전 어떤 프로젝트와 경험을 강조하거나 수정해야 하는지 구조적으로 제시한다.
- 포트폴리오 피드백 및 수정 시간을 단축한다.
- 디자이너가 자신의 강점과 부족한 역량을 명확히 이해할 수 있도록 돕는다.
- 실제 수정 행동으로 이어지는 피드백 경험을 제공한다.

---

# 3. 배경

프로덕트 디자이너는 이직 과정에서 자신의 포트폴리오가 특정 회사의 JD와 얼마나 적합한지 판단하기 어렵다. 특히 어떤 프로젝트를 강조해야 하는지, 어떤 경험이 부족하게 보이는지 명확히 알기 어렵기 때문에 지원 직전 반복적으로 포트폴리오를 수정하며 많은 시간을 사용한다.

기존 포트폴리오 피드백은 대부분 정성적이고 범용적인 조언에 머무르는 경우가 많으며, 실제 지원하려는 JD 기준으로 구체적인 수정 방향을 제시하지 못하는 문제가 있다.

또한 현재의 포트폴리오 리뷰는 대부분 사람 리뷰 기반으로 진행되며, 비용과 시간이 많이 소요되거나 피드백 품질 편차가 크다는 문제가 존재한다.

---

# 4. 사용자 문제

## 핵심 문제

- 자신의 포트폴리오가 특정 JD에 얼마나 적합한지 빠르게 판단하기 어렵다.
- 어떤 프로젝트를 강조해야 하는지 알기 어렵다.
- 부족한 경험이나 역량이 무엇인지 명확하지 않다.
- 지원 직전 포트폴리오 수정에 시간이 많이 소요된다.
- 일반적인 피드백은 많지만 실제 수정 가능한 형태의 피드백은 부족하다.
- 포트폴리오를 객관적으로 검토할 기회가 부족하다.

---

# 5. 해결 방식

## 핵심 솔루션

사용자가 포트폴리오 PDF와 채용 공고(JD)를 업로드하면 AI가 다음 과정을 수행한다.

1. 포트폴리오 내 프로젝트 및 경험 키워드 추출
2. JD 요구 역량 키워드 추출
3. 강점 및 부족 요소 비교 분석
4. 프로젝트별 보완 포인트 제안
5. 지원 직전 수정 가능한 액션 아이템 제공

- 여러 JD 기준으로 자신의 포지셔닝 방향 확인 가능
- 포트폴리오 수정 우선순위 판단 가능
- 지원 직전 빠르게 품질 점검 가능

# 10. 사용자 시나리오

주니어 프로덕트 디자이너 김아름 씨는 여러 회사에 지원했지만 서류 합격률이 낮았다. 지원 직전마다 어떤 프로젝트를 강조해야 하는지 고민이 많았고, 포트폴리오 수정에도 많은 시간이 들었다.

김아름 씨는 서비스에 포트폴리오 PDF와 지원하고 싶은 회사의 JD를 업로드했다.

서비스는 다음 내용을 분석 결과로 제공했다.

- 현재 JD와 잘 맞는 경험
- 부족하게 보이는 역량
- 강조가 필요한 프로젝트
- 수정하면 좋은 문장 및 설명 포인트
- 추가하면 좋은 데이터/성과 요소

김아름 씨는 결과를 기반으로 특정 프로젝트의 문제 정의와 성과 지표를 보강했고, 이후 다른 JD와도 비교하며 포트폴리오 방향성을 조정할 수 있었다.

# 12. 핵심 가설

## 가설 1. JD 기반 분석 니즈

사용자가 포트폴리오 PDF와 JD를 함께 업로드할 수 있다면, 일반 포트폴리오 피드백보다 JD 기준 분석 결과를 더 유용하다고 느낄 것이다.

### 검증 지표

- 분석 결과 유용성 점수 4.0/5.0 이상
- 결과 화면 평균 체류 시간 2분 이상
- 결과 저장/복사율 30% 이상

---

## 가설 2. 수정 행동 유도

분석 결과가 프로젝트별 수정 포인트와 예시 문장으로 제공된다면, 사용자는 결과 확인 후 포트폴리오를 실제로 수정하려는 의향을 보일 것이다.

### 검증 지표

- 포트폴리오 수정 의향 응답 50% 이상
- 수정 액션 클릭률 30% 이상
- 결과 복사/저장율 30% 이상

---

## 가설 3. 반복 사용 가능성

사용자가 하나의 포트폴리오를 여러 JD 기준으로 비교할 수 있다면, 지원 전략을 세우기 위해 추가 JD를 다시 업로드할 것이다.

### 검증 지표

- JD 재업로드율 15% 이상
- 2회 이상 분석 사용자 비율 20% 이상

---

## 가설 4. 신뢰 형성

분석 결과가 추상적 평가보다 구체적인 근거와 수정 예시를 포함한다면, 사용자는 AI 분석 결과를 신뢰할 것이다.

### 검증 지표

- 분석 신뢰도 점수 4.0/5.0 이상
- “피드백이 구체적이었다” 응답 50% 이상
- 인터뷰에서 재사용 의향 확인

---

## 가설 5. 유료 전환 가능성

무료 분석 결과에서 핵심 요약과 일부 개선 방향을 제공하고, 상세 리포트 또는 PDF 다운로드를 유료로 제한한다면, 사용자는 추가 결제 의향을 보일 것이다.

### 검증 지표

- 결제 버튼 클릭률 5% 이상
- 유료 상세 리포트 관심 응답 20% 이상
- 가격 수용도 조사 응답 확보

# 18. 분석 리포트 설계

## 리포트 목표

사용자가 자신의 포트폴리오를 “평가받는다”보다,
“지원 직전 무엇을 수정해야 하는지 이해”하도록 돕는다.

리포트는 다음 기준을 만족해야 한다.

- 빠르게 훑어볼 수 있어야 함
- 프로젝트 단위로 구체적이어야 함
- 수정 행동으로 이어져야 함
- 추상적 피드백을 최소화해야 함

---

# 19. 리포트 구조

## 1. 전체 요약

### 목적

지원자가 현재 포트폴리오 상태를 빠르게 이해하도록 돕는다.

### 포함 내용

| 항목 | 설명 |
|---|---|
| JD 적합 키워드 | 현재 강하게 드러나는 역량 |
| 부족 키워드 | JD 대비 부족한 요소 |
| 강점 프로젝트 | 가장 경쟁력 있는 프로젝트 |
| 우선 수정 프로젝트 | 가장 보완이 필요한 프로젝트 |
| 핵심 액션 | 가장 먼저 수정해야 할 1~2개 |

---

## 2. JD 요구 역량 분석

### 목적

JD가 실제로 어떤 디자이너를 원하는지 구조화한다.

### 예시

| JD 요구 역량 | 중요도 |
|---|---|
| 데이터 기반 의사결정 | 높음 |
| 협업 커뮤니케이션 | 높음 |
| 디자인 시스템 경험 | 중간 |
| 사용자 리서치 | 중간 |

---

# 20. 프로젝트별 분석

## 목적

사용자가 “어느 프로젝트를 어떻게 수정해야 하는지” 이해하도록 돕는다.

---

## 프로젝트 분석 구조

### 프로젝트명

커머스 리텐션 개선 프로젝트

---

### 현재 드러나는 강점

- 문제 정의 과정이 명확함
- 사용자 흐름 개선 과정이 잘 드러남
- 협업 경험이 구체적으로 표현됨

---

### 부족하게 보이는 부분

- 데이터 기반 검증 과정 부족
- 성과 지표 표현 부족
- 의사결정 근거 부족

---

### 추천 수정 포인트

| 수정 영역 | 제안 |
|---|---|
| 성과 표현 | 전환율/리텐션 수치 추가 |
| 문제 정의 | 기존 사용자 이탈 원인 보강 |
| 의사결정 | 왜 해당 UI를 선택했는지 근거 추가 |

---

# 23. 핵심 제품 방향

이 서비스의 핵심은 단순한 AI 분석 자체가 아니라,
“지원 직전 실제 수정 행동을 유도하는 피드백 경험”에 있다.

따라서 제품은 아래 방향을 우선한다.

- 분석 정확도보다 수정 가능성 중심
- 점수보다 프로젝트 맥락 중심
- 평가보다 실행 가능한 제안 중심
- 일반론보다 실제 문장 수정 중심

# 24. MVP 핵심 성공 조건

## MVP가 성공했다고 판단하는 기준

- 사용자가 결과를 보고 실제로 포트폴리오를 수정한다.
- 사용자가 다른 JD를 반복적으로 넣어본다.
- 사용자가 결과를 저장하거나 공유한다.
- 사용자가 “생각보다 구체적이다”라는 반응을 보인다.
- 사용자가 사람 리뷰 대신 1차 검수 도구로 활용한다.
